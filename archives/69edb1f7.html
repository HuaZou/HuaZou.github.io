the principal of running shiny apps

用户通过web或app选择参数，这些参数输入到UI再到后台的server运行，得到的output输出到UI，最后再返回到web或app。该传输链达成一个完整的用户与server的交互使用效果。

Shiny app的组织结构

shiny app主要由三部分组成：

  1. 定义交互式界面的对象(ui.R)

  2. 服务器运行的函数

  3. 启动app的shinyApp函数

ui.R

ui.R包含了定义 page-> Panel -> sidebarLayout -> sidebarPanel -> siderInput -> mainPanel. 从页面到参数输入最后到可视化结果。

Notes: shiny version 0.10.2后可以将ui.R和server.R合并成一个R脚本app.R。

server.R

server主要是在服务器运行的R函数，其输入参数来自ui.R的inputs参数，我们能发现：

  1. input是ui.R的sliderInput函数的input参数bins；

  2. 输出结果是用renderPlot，这是因为最后的结果是图形；数据则是reactive.

  1. 在Rstudio打开app.R脚本；

  2. 在右上角Run App选择 Run in Viewer Pane

Cheatsheet

Rstudio官方提供了shiny的cheatsheet，主要由七部分组成

basics

shiny web或app如何运行以及它们的架构是什么(app.R或ui.R/server.R)，最后还给出了如何部署它们，个人倾向与使用shiny server部署。

Building an app

构建APP需要输入输出函数，在ui函数也即使用户选择参数提供输入，该参数作为server函数的输入，再运算完成后作为输出结果返回给ui的output函数，其参数来自server的输出结果

inputs

inputs作为ui函数用户选择的参数，其范围包含actionButton, fileinput, checkboxGroupInput和passwordinput等

outputs

outputs是render*() 和 *Output()两大函数组成，它们将server的输出结果输入到ui。

Reactivity

每次用户更改ui参数都可能诱发server运行脚本，可以通过设置某些isolate隔离不必要的重复运行的命令

UI design by HTML language

UI可以通过HTML语言进行设计。

Layouts

除fluidPage函数外，还有很多如fluidRow flowLayout等等函数可以组合使用，从而调整ui界面。
