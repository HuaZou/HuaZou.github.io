A brief data analysis in single-cell RNA sequencing

Posted on 2020-11-12 In OmicsData , single cell Views: Valine:
Symbols count in article: 9.8k Reading time ≈ 9 mins.

Workflow

This post will focus on Data normalization and Cluster analysis. Data normalization consist of calculating the Mitochondrial counts ratio and the nFeatures counts, normalizing the filtered data and performing principal component analysis (PCA) on these data to Dimension Reduction. Cluster analysis is based on the significant PCs to figure out the different cell types.

​

Data source

You can retrieve the raw data whose format was h5 by using the GEO number GSE147863 in the Single Cell Human Kidney Organoid Analysis.

Library packages

load data

  • h5 file is the one of the output files of CellRanger (a single cell RNA-seq workflow from the raw fastq data to counts)

The alternative inputs could be three parts: Read10X(data.dir = path)

  1. barcodes.tsv.gz;
  2. features.tsv.gz
  3. matrix.mtx.gz

Create Seurate Object

metadata in Seurate Object: KidneyOrganoid@metadata

  1. orig.ident: this often contains the sample identity if known, but will default to project as we had assigned it
  2. nCount_RNA: number of UMIs per cell
  3. nFeature_RNA: number of genes detected per cell

quality control

  • nFeatures per cell: double cells in one samples have more features and higher counts
  • nCounts perl cell
  • the percentage of Mitochondrial counts per cells: the death cells maybe have higher MT RNA ratio

Add %MT as quality control metrics

quality control metrics Plots

Result

  1. the range of nFeatures: [0, 8000]
  2. the MT ratio is too large. In fact, plenty of tissues’ cells couldn’t contains lots of MT RNA except Heart’s cells.

Get quality control Thresholds

Result: choosing the points in the 1st and 4th quartile range as the remained cells

Quality control plots

Filtering based on the quality parameters

subset function is used to choose the subpopulation of the whole data

Counts normalization

Before performing clustering, we need to normalize our high quality scRNA data to remove many other factors. the main factors in data normalization are sequencing depth and gene length.

Cell cycle effect

Result: The dataset has large differences due to cell cycle phase. Finally, we need to regress out the variation due to the cell cycle

SCTransform

the SCTransform is a method of normalizing, estimating the variance of the high quality data and identifying the most variable genes. It requires more resources of server memories.

Principal Component Analysis

Performing PCA based on the most variable genes

Clustering

  • the metagenes : principal components
  • the Shared Nearest Neighbor (SNN) Graph

Clustering cells based on top PCs (metagenes)

Result: we can see the differences of each selected PCs with the most variant genes and the next step is to find the top 10 most variable genes per PC

the elbow plot on selecting PCA Components

Result: the parameter for PC components is 20 dimension

Cluster analysis

using the SNN graph to find the neighbors and then cluster all the data in a unsupervised way. The resolution parameter will determine the number of cluster, we also choose various resolutions to get the suitable result

UMAP visualization

Result: 17 clusters have been found in this dataset, but it was different compared to the original result from the article. In fact, the author reclustered the data after identifying the cell types.

Feature Plots on interesting genes

Find Cluster Markers

Result: the gene markers per cluster may contribute to identify the cell types. pct.1 means the markers’ ratio in their own cluster, and pct.2 means the ratio in the rest of cluster. pct.1 minus pct.2 is the difference value.
