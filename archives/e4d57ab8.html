从Ribo-seq reads识别CDS序列：PRICE pipeline

Posted on 2020-11-21 In OmicsData , Ribo-seq Views: Valine:
Symbols count in article: 10k Reading time ≈ 9 mins.

Ribo-seq

Ribo-seq是基于Ribosome footprinting的技术，它在50年前就出现了，但受制于当时技术，无法实现real-time观测mRNA转录和大规模地成功提取细胞内正在转录的核糖体。

实现步骤

  1. We start by lysing the cells by first flash-freezing them, and then harvesting them in the presence of cyclohexamide (see explanation for this under ‘Drawbacks and complications’).
  2. Next, we treat the lysates with RNase 1, which digests the part of the mRNA not protected by the ribosome.
  3. The ribosomes are then separated using a sucrose cushion and centrifugation at very high speeds.
  4. RNA from the ribosome fraction obtained above are then purified with a miRNeasy kit and then gel purified to obtain the 26 – 34 nt region. These are the ribosome footprints.
  5. From there, the RNA is dephosphorylated and the linker DNA is added.
  6. The hybrid molecule is then subjected to reverse transcription into cDNA.
  7. The cDNA is then circularized, PCR amplified, and then used for deep sequencing.

Ribo-seq vs. RNA-seq**

通过结合Ribo-seq 和RNA-seq，有研究者证实mRNA富集并不一定意味着翻译得到的对应蛋白质也多。前者可以测量mRNA的翻译效率，也让我们认识到 gene -> mRNA -> protein存在较多的不确定性。在比较了蛋白质与Ribo-seq和RNA-seq的相关性后，结果揭示蛋白质和Ribo-seq有更高的相关性，这也说明可能使用Ribo-seq表征基因表达可能效果会更加。

New insights in Ribosome profile

  1. 某些先前认为是短的非编码序列也存在翻译的可能，但其功能未知；
  2. detection of previously unknown translated short upstream ORFs (uORFs), which often possess a non-AUG start codon. These uORFs are likely responsible for regulating the protein-coding ORFs (which is true of the GCN4 gene), though it remains to be seen if that is true for all uORFs or if they have other currently unknown functionalities;
  3. 测量出大概的翻译效率（330 codes per minute）;

该技术从翻译水平研究基因表达情况，这是转录水平无法比拟的优势。翻译水平和蛋白质联系更紧密，可以更容易解析表型上的差异，如不同疾病的形状差异是否与基因的翻译水平有关呢？

Drawbacks and complications

如何获取细胞内部的Ribosome RNA是该技术最大的难点之一。

The best method of combatting these artefacts is to flash-freeze the cells prior to harvesting, lyse over them dry ice, and then continue the protocol in the presence of cyclohexamide.

PRICE

PRICE是采用Expectation-Maximization算法计算密码子活动概率从而识别核糖体测序数据内的开放阅读框架的方法。该方法是Gedi软件的插件之一。PRICE包含了鉴别和评分密码子以及预测Ribo-seq reads的ORF信息的步骤（数据过滤、比对和可视化等功能）。

PRICE (Probabilistic inference of codon activities by an EM algorithm) is a method to identify ORFs using Ribo-seq experiments embedded in a pipeline for data analysis.

Gedi is a software platform for handling genomic data such as sequencing reads, sequences, per-base numeric values or annotations written in Java.

相比先前把来自某类特定长度所有的Ribo-seq reads（如 29bp长度的reads）比对到某个特定的位置上得到ORFs的传统方法，这类方法依赖先验知识即已知长度的reads类和位置信息。PRICE是利用最大期望算法对所有reads进行计算，推断激活的转录密码子，基于推断密码子信息，候选ORF可通过广义二项分布模型预测得到。

安装PRICE

  • PRICE是基于JAVA语言的软件，因此需要安装JAVA，官方指定版本是JAVA 1.8: Centos 8下安装

  • 安装R，PRICE具有可视化功能，可以通过conda安装R

  • 下载最新release的Price压缩包，然后解压即可使用

  • 其他可能需要用到的软件

  • Using PRICE

必须参数：输入reads一般是BAM文件

  • -reads: Can either be a single BAM file containing mapped reads (make sure that you mapped reads in end-to-end mode, otherwise the information of 5’-end mismatches is lost; also, PRICE needs the MD attribute to identify mismatches, so make sure it is present in the BAM file!), a *.bamlist file containing names of BAM files (one per line), or a CIT file (e.g. as produced by the pipeline).
  • -genomic: one or several (separated by space) names of prepared references; make sure that these are the same as used to map the reads!
  • -prefix: A user chosen prefix for all output files (e.g. -prefix price/hsv)

其他参数：

  • -progress: Show the progress interactively on the command line
  • -plot: Generate several plots (PRICE model, signal-to-noise ratio plot)
  • -nthreads: Set the number of parallel threads to run PRICE (default: use all cores)
  • -delta: Set a regularization parameter (default is no regularization)
  • -inferDelta: Infer the regularization parameter such that that a 10% off-frame codon is recognized as such
  • -novelTranscripts: Use mapped reads to infer novel transcripts (important for not so well annotated organisms, or when only CDS are annotated)
  • -fdr: set the desired false discovery rate (default: 10%)
  • -percond: Infer the codon model for each processed sample individually (instead of one common model for all samples of the experiment)

结果解析

输出结果：

  • ${prefix}.param: A list of all parameters used to call PRICE
  • ${prefix}.orfs.tsv: A table (e.g. to import into Excel) containing all the information about all called ORFs (before filtering!)
  • ${prefix}.orfs.cit: A CIT file containing the ORFs filtered by FDR (default: 10%) (to extract ORFs quickly using ViewCIT and used by the viewer)
  • ${prefix}.orfs.cit.metadata.json: The associated metadata of the CIT file (condition names, total read numbers, etc.)
  • ${prefix}.estimateData: table containing the minimal sufficient statistics for estimating the PRICE model
  • ${prefix}.model: the PRICE model in binary format
  • ${prefix}.codons.cit: A CIT file containing all called codons
  • ${prefix}.*.rmq: Several index files to efficiently show codon activities in the viewer.

orfs.tsv 文件包含单个样本reads的所有ORFs信息。
