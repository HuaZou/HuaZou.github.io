git分支保存hexo博客源码

发表于 2020-04-03 分类于 工具 ， Hexo 阅读次数： Valine：
本文字数： 1.8k 阅读时长 ≈ 2 分钟

hexo撰写博客流程

  1. 先跳转到博客目录下面，再在powershell或者item等命令行终端输入hexo new post_name命令，会自动在source/_post目录下生成post_name.md文件；
  2. 使用Typora软件编辑该md文件，再在命令行终端输入hexo g生成HTML文件，如果需要更新该md文件，则需要使用hexo clean删除对应HTML文件后再使用hexo g命令；
  3. 在发表前，可以在本地使用hexo s命令预类渲染结果，如没有问题后可通过hexo d命令发布到远程仓库master分支。

源码保存问题

在master分支只会保存博客的静态网页，无法保存源码。由于每个人可能拥有两台以上的终端，即使使用云盘备份也会很麻烦（比如本人使用OneDrive个人版保存源码，结果因为时常更新静态文件，导致源码一直在同步，这是及其不友好的）。另外，时常需要对博客更新某些功能，这又导致源码经常会修改，而修改记录又无法保存。最严重的问题是，一旦某台终端硬盘损坏，这可能会导致源码完全丢失。git分布式管理系统的分支策略设计完美解决了这一痛点。

解决办法

创建source分支

在github的username.github.io仓库新建source分支，红色箭头处输入新分支名字（我已创建source分支）

更改默认分支

把默认的master主支切换成 source分支，该步骤在settings->Branches->Default branch，将默认分支由master更改成source，最后update。博客静态网页是发布到master主支，主支不是我们应该关心的问题。

克隆source分支

  1. 克隆hexo分支到本地, git clone git@github.com:HuaZou/HuaZou.github.io.git；
  2. 在终端中cd进入该username.github.io文件目录,cd username.github.io；
  3. 在当前目录使用git branch`命令查看当前所在分支，应为新建的分支source.

上传源码文件

  1. 先将本地博客的源码文件（Hexo博客项目目录下的全部文件）全部拷贝进username.github.io文件目录中去；

  2. 准备将所有的文件都提交到source分支，提交时考虑以下注意事项：

    • 将themes目录以内中的主题的.git目录删除（如果有），因为一个git仓库中不能包含另一个git仓库，否则提交主题文件夹会失败;
    • 可能有人会问，删除了themes目录中的.git不就不能git pull更新主题了吗，很简单，需要更新主题时在另一个地方git clone下来该主题的最新版本，然后将内容拷到当前主题目录即可

  3. 修改.gitignore文件，不建议上传一些大文件和以及个人隐私博客等，这些文件可以保存在云端；

  2. master主支和source分支各自保存着一个版本。master主支用于保存博客静态资源，提供博客页面供人访问；source分支用于备份博客部署文件，供自己维护更新，两者在一个GitHub仓库内也不会有任何冲突。

同步至其他终端

添加ssh key至GitHub账户，克隆仓库，按照hexo等。在任何终端对博客做任何修改前先git pull更新最新的文件。
